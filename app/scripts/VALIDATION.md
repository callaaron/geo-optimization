# GEO 引擎 B2B 可用性验证报告

> 验证脚本：`scripts/validate.ts`（纯文本路径，esbuild 打包 + Node 运行，不触发 DOMParser）
> 运行：`node_modules/.bin/esbuild scripts/validate.ts --bundle --platform=node --format=cjs --alias:@=./src --outfile=/tmp/validate.cjs && node /tmp/validate.cjs`
> 结论：**全部断言通过 ✅**，并已修复 3 处会导致 B2B 不可信的评分缺陷。

## 1. 测试设计

分层样例（真实风格中文 B2B 文案，质量梯度）：
- `STRONG` 工业 MES 产品页（型号/规格/案例/认证/报价预约）
- `MEDIUM` 公司方案页（有实体有数据，缺案例与转化引导）
- `WEAK` 空泛「关于我们」（套话、无数据、无实体）
- `COMP1` 数据集成平台竞品（强）
- `COMP2` 协同办公平台竞品（中偏弱）

每个样例分别跑 `general` 与 `b2b` 模式；并对「我的站点 + 2 竞品」跑 `benchmarkSites({mode:"b2b"})`、`aggregateSite`、`buildReport`。

## 3. 发现的缺陷与修复

### 缺陷 A（严重）：EEAT 评分倒置
- 现象：强 MES 页 EEAT=0，而弱协同平台页因含「文档」二字 EEAT=25。
- 根因：`scoreEeat` 的署名正则 `/作者|文\/|文|by /i` 中裸 `文` 命中「文档」「文化」等；且文本模式下无外链/元数据/作者，唯一内容信号 `CREDENTIAL` 未覆盖「认证/专利/资质」，导致真实权威文案得 0 分。
- 修复：署名正则收紧为 `/(作者\s*[:：]|文\/|by\s+[a-z一-鿿]|author\s*[:：])/i`；`CREDENTIAL` 扩充 B2B 权威/资质词（认证、专利、专精特新、高新技术、ISO、检测报告、行业龙头、500强、首席、合伙人等），并将资质分改为随命中数缩放（15→30）。

### 缺陷 B（中等）：技术可读性恒为 75
- 现象：所有样例（含空输入）技术维度均 =75，与内容无关。
- 根因：`scoreTechnical` 在纯文本模式 `else { s = 75 }` 硬编码。
- 修复：纯文本模式改用「结构/可读性代理」——段落数（≤6→35）、平均段长合理度（25）、标题层级数（≤4→25）、字数充足度（15）综合评分。修复后强页 93、弱页 69、竞品2 61，具备区分度。

### 缺陷 C（中等）：新鲜度恒为 25
- 现象：所有样例时效均 =25。
- 根因：`extractDates` 仅匹配「YYYY年MM月」等含月格式，裸年份「2016 年」「2019 年」无法识别，故 `p.dates` 恒空 → `scoreFreshness` 走 25 兜底。
- 修复：`extractDates` 增加 `YEAR_RE`（仅 1990–2029，降误报）抽取裸年份；`newestDate` 支持解析纯年份；时效分级改为「无日期=30 / 近期=100 / 2 年以上已知陈旧=15（低于无日期默认，体现陈旧惩罚）」，并加「最新/近期/今年」等显式时效词 +12 加权。修复后 MEDIUM(2016)=15、COMP2(2019)=15，确实被识别为陈旧。

## 4. 修复后的真实性检验
- benchmark 建议从「我方 EEAT 落后竞品 25 分」（错误，源自缺陷 A 倒置）变为「我方在各维度均不落后于竞品，可聚焦强化优势」——与事实一致（STRONG 确为对标组最高分）。说明修复后建议生成逻辑可信。

## 5. 已知限制（非缺陷，设计如此）
- **结构化数据维度在文本模式下恒为 0**：JSON-LD / OG 标签只能从 HTML/URL 抓取获得；粘贴纯文本时无法承载，UI 已提示「发布为网页后可获取完整诊断」。
- **实时 URL 抓取受沙箱网络限制**：公共 CORS 代理可能被目标站拦截，此时 UI 引导改用「粘贴内容」，不阻塞分析。
- 体积：打包后 JS ≈ 762KB（gzip 227KB），有 chunk 体积提示，可后续按需 code-split，不影响功能。
