## 3. 真实 AI 改写结果（`/api/ai/rewrite`，mode=b2b）

### 3.1 后端接口调用结果：❌ 失败（重要缺陷）

按任务给定的标准命令连续调用 **5 次**，均返回：

```json
{"ok":false,"error":"模型未返回可解析的 JSON"}
```

对比验证：`/api/ai/analyze` 接口使用同一模型、同一 Key、同一 Agent Plan 路径 **调用正常并返回合法 JSON**，说明问题与"Key/网络/模型不可用"无关，而是**该改写接口的特有问题**。

### 3.2 根因诊断（已定位）

通过复用后端同一 `ark.mjs` 客户端与同一 prompt 直接抓取原始模型输出，确认：

- AI 改写 prompt 要求输出**长 Markdown 全文 + FAQ + 定义表 + changes**，输出体量大；
- 后端 `ai.mjs` 中 `aiRewrite` 设定 `maxTokens: 2600`，**不足以容纳完整 JSON**；
- 模型输出在 `definitions` 数组中途被**硬截断**，原始输出末尾为：

  ```
  ...{"term":"
  ```
  即 JSON 被切断在字符串内部，`parseJsonLoose` 即便裁剪到最后一个 `}` 也无法补齐，最终抛出"模型未返回可解析的 JSON"。

**结论**：这是 `aiRewrite` 的 **token 预算过低导致的截断型 bug**，非内容或模型能力问题。将 `maxTokens` 提升至 **4096** 后，同一 prompt 即返回**完整且可解析**的结果（见 3.3）。

| TL;DR | 与正文首句完全重复，未压缩 | 提炼为「B端贴牌代工供应商」精准定位 |
| 结构 | 平铺，仅 TL;DR+数据+1 定义+正文 | TL;DR + 关键要点 + 6 层正文 + FAQ + 询价路径 |
| FAQ | 1 条，且为标题复述，无信息量 | 6 条（OEM、资质、产品、工艺、品牌、询价），B2B 导向 |
| 定义/实体 | 1 条且**误判**（"位于山东青岛"） | 7 条准确（公司/许可证/海鲜水饺/鱼糜/OEM/荣誉/关联企业） |
| 事实保真 | 100% 保真（机械搬运） | 100% 保真，未编造 |
| 缺失数据处理 | 无 | 用【待补充：…】占位（联系方式、箱规等），符合预期 |
| 可落地性 | 可作"保真骨架/兜底" | **更适合正岛食品直接落地** |

**哪边更适合正岛食品直接落地？**
→ **AI 改写明显更适合**。它输出了 B2B 客户真正会问的结构（资质编号、OEM/ODM、批发、询价路径），并补齐了实体定义与 FAQ，显著提升被 ChatGPT/豆包/Perplexity 直接引用作答的概率。规则改写仅适合作为"零成本、零延迟"的兜底骨架。

**AI 是否出现编造？**
→ **未编造**。对原文未提供的联系方式（电话/邮箱）、具体规格（单包克重/箱规）、详细地址，模型一律用 `【待补充：…】` 占位，未杜撰任何数字或名称。这一点符合 GEO「不编造」要求，是正岛食品可放心使用的特性。

**规则改写暴露的 bug（需在模块侧修复）：**
1. 定义抽取正则把地名中的"即"误判为释义标记 → 错误定义。
2. FAQ 把整句标题当问句 → 无信息量。
3. TL;DR 与正文重复，无压缩提炼。

## 5. 测试结论

### 模块可用性
| 模块 | 可用性 | 说明 |
|------|--------|------|
| 规则改写引擎（rewriter） | ✅ 可用（质量有限） | 无报错，输出 JSON 完整；但结构弱、含 1 处定义误判、FAQ 无信息量 |
| 结构化数据生成器（generator） | ✅ 可用 | llms.txt / JSON-LD(Article) / robots.txt / metaTags 均正常产出，可部署 |
| 真实 AI 改写（/api/ai/rewrite） | ❌ 当前不可用 | 因 `maxTokens=2600` 导致输出截断、JSON 解析失败；同类 analyze 接口正常，故为可修复的 token 预算 bug |

### 对正岛食品的主要价值
1. **生成器可直接产生 llms.txt + JSON-LD + robots + meta**，让正岛官网具备"被 AI 引擎结构化读取"的基础能力，是 GEO 落地成本最低的一步。
2. **AI 改写（修复后）能产出 B2B 友好的高引用度内容**：资质编号、OEM/代工、批发、询价路径、FAQ、实体定义一应俱全，且对缺失信息严格用【待补充】占位，不编造——非常适合作为正岛食品对外招商/供页的内容底稿。
3. **规则改写可作零成本兜底**：在网络/Key 不可用时仍能给出保真骨架。

### 主要不足与建议
1. **【高优·后端 bug】** `aiRewrite` 的 `maxTokens` 应从 2600 提升至 ≥4096，否则对中等篇幅官网文案必然截断失败。
2. **【生成器】** JSON-LD 的 `keywords` 为碎片化拆词（"制品""公司成立"），建议改为实体/产品词表抽取；`publisher.name` 应填公司名而非整条标题。
3. **【规则引擎】** 定义抽取正则需排除地名中的"即"等假释义标记；FAQ 生成应基于真实问句模式而非整句标题；TL;DR 应做压缩而非复制首句。
4. **【内容侧·正岛食品】** 原文缺失联系方式、产品规格（克重/箱规）、公司详细地址，AI 已用【待补充】标记，建议补真实信息以进一步提升被引用率。
